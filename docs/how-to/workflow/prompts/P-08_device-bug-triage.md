# P-08: 実機固有バグのトリアージ

- **渡す先**: Phase 1 → Claude Code / Phase 2 → Codex
- **タイミング**: P-06 の実機検証で問題が見つかった後
- **目的**: Claude Code がログを解析して根本原因を特定し、Codex に具体的な修正箇所を指示する

## Phase 1: Claude Code による解析

### 1-1. ログ分析

P-06 で取得した以下を読み込む:

- logcat ログ（`adb logcat` 出力）
- クラッシュログ（`adb logcat -b crash`）
- メモリ情報（`dumpsys meminfo`）
- スクリーンショット

## Phase 2: Codex への修正指示

Phase 1 の結果を基に、P-03 形式で Issue を起票する。

### Issue テンプレート

```markdown
## Bug: #{番号} {バグの概要}

### 現象

{P-06 で確認した現象。再現手順を含む}

### 根本原因（Claude Code が特定済み）

{Phase 1-4 の結果をそのまま記載}

### 修正方針（具体的な変更箇所）

| ファイル                        | 行番号    | 変更内容       |
| ------------------------------- | --------- | -------------- |
| `{PROJECT_ROOT}/src/{ファイル}` | L{行番号} | {具体的な変更} |

### AC

- [ ] {修正が正しいことの確認条件}
- [ ] 既存テストが全パス
- [ ] 修正箇所のユニットテスト追加
- [ ] `pnpm verify` 全パス

### 読むべきファイル（絶対パス）

| ファイル                                                 | なぜ                  |
| -------------------------------------------------------- | --------------------- |
| `{PROJECT_ROOT}/src/{該当ファイル}`                      | 修正対象              |
| `{PROJECT_ROOT}/docs/reference/tasks/lessons.md`         | 同種バグの過去事例    |
| `{PROJECT_ROOT}/docs/how-to/workflow/git_workflow.md`    | ブランチ命名・PR 手順 |

### Codex への注意

- この問題は**実機でのみ再現する**
- ログの該当行: `{具体的なログ行を引用}`

- 方針に疑問がある場合は実装せず質問すること
```

## 担当分担まとめ

| 担当            | やること                                                          |
| --------------- | ----------------------------------------------------------------- |
| **Claude Code** | ログ読み → 仮説 → 検証 → 根本原因特定 → 修正方針決定 → Issue 起票 |
| **Codex**       | Issue の修正方針に従いコード修正 + テスト追加 + PR 提出           |

## 修正後の確認

Codex の PR がマージされたら:

1. P-05 で再ビルド
2. P-06 で同じチェックを再実行
3. 問題が解消されていることを確認
4. 教訓を `{PROJECT_ROOT}/docs/reference/tasks/lessons.md` に追記
