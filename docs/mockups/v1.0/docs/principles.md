## 3 行宣言

1. **BonsaiLog は事実だけを記録します。**
2. **アプリは命令しません。判断はユーザーがします。**
3. **目安と参考だけ示し、判定はしません。**

---

## なぜこの原則があるのか

競合 7 アプリ（Planta / Greg / PictureThis / Bonsai Album / Bonsai Care App / Mirai Mobile / Appy Bonsai）の最大の不満は **「アプリの言うことが間違っている」** だった。
BonsaiLog は **「教えない、正しく記録する」** を選び、市場で唯一の差別化を獲得する。

哲学はすでにコードに宿っている：

- `wireframes/home-screens.jsx:329` のコメント
  「直近の事実1件のみ。推奨や状態判定は出さない（高橋・プロのNG回避）」

本ドキュメントはそれを全機能に拡張したもの。

## Pro 機能の方針

**Pro = データ制限解除ではない。Pro = 利用頻度 gating ＋ 出力 ＋ 体験。**

> **Pro 機能 6 項目の境界 SoT**: [ADR-0049](../../../adr/ADR-0049-pro-feature-boundary-v1.md) (Sess58 確定、 Sess59 実装着手)。 v1.9 までの「Pro = 出力 ＋ 体験 (2 軸)」 方針は Sess58 議論で「**Pro = 利用頻度 gating ＋ 出力 ＋ 体験 (3 軸)**」 へ拡張された (業務利用層獲得 + Day One モデル参考)。

| 種類                                                | Free                            | Pro                                  |
| --------------------------------------------------- | ------------------------------- | ------------------------------------ |
| 樹木登録数                                          | 無制限                          | 無制限                               |
| ① 基本情報写真                                      | **3 枚/本まで** (ADR-0049)      | 無制限                               |

## App Store マーケコピー方針 v1.2

**方針**：哲学を売りに転換するが、過剰にならない。**機能訴求 ＋ 哲学声明** の二層構成。

### 階層 1 ─ メインメッセージ（機能訴求・誰でもわかる）

> あなたの盆栽を、毎日記録するアプリです。

- 写真、作業、メモ、樹形、樹種、すべて自分のもの。
- 1 株でも 150 株でも、書き出しも検索も自由に。

### 階層 2 ─ 哲学声明（差別化・補足）

> BonsaiLog は事実を記録します。アプリは命令しません。判断はあなたが行います。

- 「水やりの時期です」「枯らさないように」のような<em>判定</em>はしません。
- 提示するのは「目安」と「参考」だけです。
- 通知はあなたが決めた予定のときだけ届きます。

### 競合差別化の言語化（任意で末尾に）

> 他のアプリのように「アプリの言うことが正しい」とは言いません。
> あなたの記録があなたの教科書になります。

### NG コピー（App Store 説明文から除外する）

- 「枯らさない」「失敗しない」「健康診断」「AI が判定」「水やりを忘れない」
- 「自動でお知らせ」「最適なタイミング」「あなたに最適な」
- 数値捏造：「精度 99%」「30 日で習慣化」「失敗率 0%」

### 採用理由（議論記録）

- **判断**：v1.2 議論（2026-05-07）で **a 案（折衷）** 採用。「判定しないことの強さ」を売りにしつつ、過剰にならないよう機能訴求を上に置く二層構成。
- **比較**：b 案（機能訴求のみ）はリスク低いが、Planta/Greg/PictureThis との差別化がぼやけ、v1.1 哲学全面適用の意義が伝わらない。
- **規制的セーフ**：FDA AI/ML 規制強化（2025）と App Store Guideline 1.4.1 medical scrutiny 強化の流れで、判定型アプリは規制リスクが上昇。「判定しない」は規制セーフでもある。

## ATT/UMP 統合計画 v1.2

**方針**：19 言語ターゲット運用のため法令対応必須。<b>ユーザー負担最小化</b>のため pre-prompt（価値訴求） → OS ダイアログの順で、地域判定で EU/UK/CH のみ UMP 表示。

### 対象法令

- **Apple ATT**（App Tracking Transparency） ─ iOS 14.5+ 必須。広告 ID 利用に同意が必要。
- **UMP TCF v2.3**（Google User Messaging Platform / IAB TCF v2.3） ─ 2026-02 経過済義務化。EU/UK/CH 配信時に必須。未対応で <b>EEA Limited Ads = 収益制限リスク</b>。

### Onboarding FLOW 再構成案

```
旧（v1.1）: 価値紹介 → 通知許可 → 完了
新（v1.2）: 価値紹介
            ↓ （地域判定）
           [EU/UK/CH のみ] UMP 同意画面
            ↓
           ATT pre-prompt（「広告の質を保つために許可をお願いします」価値訴求）
            ↓
           ATT 標準ダイアログ（OS）
            ↓
           通知許可（任意・OFF 既定）
            ↓
           完了
```

### 実装方針（ユーザー負担最小化）

| 観点                  | 実装                                                                                                                          |
| --------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| pre-prompt の価値訴求 | 「広告の質を保つために許可をお願いします」（Apple HIG 推奨）／ NG「すべての機能を使うために許可してください」（強制感がある） |
| CTA                   | 「次へ」＋「後で決める」を併設（完了率の極端な低下を防ぐ）                                                                    |
| 拒否時の挙動          | 全機能引き続き使用可（Apple HIG 1.6.4「manipulate しない」整合）                                                              |
| 地域判定              | デバイスロケール ＋ IP（保守的に EU/UK/CH 表示）                                                                              |
| iOS 14.5 未満         | ATT スキップ（自動判定）                                                                                                      |
| 19 言語対応           | i18next 化（P3-1）と同時実装                                                                                                  |

### 既存資産（実装済の関数）
