# ADR-0060: ja 文言スタイル基準の確立と全キー監査 (ja 先行スコープ)

- Status: Accepted
- Date: 2026-06-12
- Deciders: @doooooraku
- Related: Issue #1208 / #1207 (翻訳負債) / ADR-0033 (i18n 翻訳ポリシー) / R-3 (品の良さ) / `docs/reference/copy-style-ja.md`

---

## Context（背景：いま何に困っている？）

- 現状: ja.ts は 875 キーまで成長したが、文言は機能 PR 単位の書き起こしで、照合する文体基準が存在しなかった。
- 困りごと: 表記揺れが実在する — 同一 paywall 画面内で「買い切り」と「買切」が混在、英数字前後スペースの有無混在 (`Proプラン` vs `Pro に`)、空状態の句点有無混在。揺れは品質印象を損ない、#1207 (17 言語翻訳) 着手時の翻訳指示も不安定になる。
- 制約/前提:
  - 禁止語 (R-3 / constraints 第5章) は既に機械検査済みだが、文体・表記のルールは未定義・未検査だった。
  - Sess102 決定 (#1207): 翻訳着手時は「① ja (SoT) 文言の表現見直しが先 → ② 言語別ペルソナ翻訳」。本 ADR は ① の前倒し実行。

## Decision（決めたこと：結論）

- **文体・表記基準を `docs/reference/copy-style-ja.md` として成文化し、機械検査可能な項目を `pnpm i18n:style` (`scripts/i18n-style-ja.mjs`) で lint する。**
- **スコープは ja.ts のみ先行**。en・他 17 言語・fastlane metadata は対象外 (他言語は #1207 台帳のトリガ待ち、en は意味乖離が問題化した時点で再判断)。
- 基準の骨子 (詳細はガイドが正): 説明文 = です・ます調 + 句点 / ラベル = 体言止め + 句点なし / 英数字前後半角スペース / 半角記号 / 表記揺れ辞書 / 専門用語維持。
- 適用範囲: v1.x 全画面のユーザー可視 ja 文言。

## Decision Drivers（判断の軸）

- 再発防止: 修正だけでは次セッションで揺れが再蓄積する。基準 + lint + 採用ゲート (mechanism 3 点セット) で構造的に止める。
- 翻訳資産性: ja 確定 + 文体指針は #1207 の前提作業として二重に回収できる。
- ユーザー像: 高橋ペルソナ (62 歳) には丁寧さ = 安心。削るのは冗長表現であって敬体ではない。

## Consequences（結果）

- Pro: 新規キーが追加時点で基準準拠になる (`/i18n-add` Skill + verify チェーン)。揺れの再蓄積が止まる。
- Con: ja と en の文体最適化が非対称になる (en は未監査のまま)。意味を変える修正をした場合の en 同期は将来課題として #1207 に委ねる。
- 運用: lint は導入時 warn → 全件監査完了後に error 昇格 + `verify:i18n` 組込 (Issue #1208 PR-6)。

## Notes

- 課金系文言 (Purchase / Paywall) は意味不変を原則とする — Console IAP 説明文・ADR-0049 との整合を壊さない。
- Maestro E2E は日本語文言に依存していないことを確認済み (.maestro/ 内 text: 日本語ヒット 0 件、2026-06-12)。
