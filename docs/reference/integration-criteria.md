# 整合性レベル定義 — 「整合済」マークの根拠

> **このファイルのロール**: ADR-0020 §画面マップ で使われる「整合済」マークの **要求水準** を明文化する単一情報源。Claude Code / 開発者が PR で「整合済」と判定する際の客観的基準。

最終更新: 2026-05-10 (T1-1、Tier 1a 基盤整備)

---

## 背景 — なぜこのファイルが必要か

本セッション (2026-05-10) で **PR #341/#342/#350 で「整合済」マーク** したが、実機画像比較で **チェックマーク overlay / SelectionToolbar / FAB 連動 / 「N件選択中」表示** の乖離が発覚。

**根本原因**: 「整合済」の定義が曖昧で、Claude Code が **構造的整合 (component 階層が同じ)** で「整合済」マークしていたが、ユーザー期待は **見た目 80% 以上の整合** だった。

→ 本ファイルでレベル定義を明文化、PR で参照する。

---

## 整合性レベル定義 (4 段階)

### レベル 0: 構造のみ (ふんわり整合)

- **判定基準**: component 階層が mockup と同じ
- **見た目**: 大きく異なる (要素サイズ、色、配置が違っても OK)
- **動線**: タップ動作が違う可能性あり
- **PR マーク**: 「**部分整合 (構造のみ)**」と明記、整合済とは言わない
- **例**: PR #341 BonsaiCard チェックボックス追加だが、実機で表示されていない (構造はある、見た目に出ていない)

### レベル 1: 動線整合

- **判定基準**: 構造 + タップ/長押し動作が mockup と同じ
- **見た目**: やや異なる (写真サイズ、フォントが違う可能性)
- **動線**: ユーザー操作感は mockup と一致
- **PR マーク**: 「**動線整合済 (見た目要再確認)**」

### レベル 2: 見た目 80% 一致 ⭐ (整合済の **最低水準**)

- **判定基準**: 主要要素 (写真サイズ / フォント / 色 / 配置) が一致
- **許容**: 細部 (角丸 px ずれ、padding 1-2px ずれ、shadow 強度) は許容
- **必須一致項目**:
  - [ ] 写真/画像のサイズ (mockup と同じ縦横サイズ)
  - [ ] フォント (NotoSerifJP_500 / NotoSansJP / IBMPlexMono 等の正しい使用)
  - [ ] 色トークン (BRAND_GREEN / BG_PRIMARY / BG_SURFACE / TEXT_PRIMARY 等)
  - [ ] 主要レイアウト (3 階層 / 2 階層 / Hero + Tabs 等の構造)
  - [ ] 主要ボタン/CTA の配置 (画面上下)
- **判定方法**: 視覚比較 (mockup スクショ vs 実機/Web スクショを Read で並べて目視)
- **PR マーク**: 「**整合済 (レベル 2)**」 ← **当面の「整合済」要求水準**

### レベル 3: 見た目 95% 一致 (将来目標)

- **判定基準**: 細部 (角丸 / padding / shadow / 余白) も mockup と一致
- **許容**: pixel parity 近い、ほぼ完全コピー
- **判定方法**: Visual regression testing (Storybook + Chromatic / Percy / ui-diff Web 版)
- **PR マーク**: 「**完全整合 (レベル 3)**」
- **適用時期**: Tier 3 (Visual regression 導入後)

---

## 「整合済」マーク要求水準 (運用ルール)

### 当面 (Tier 1〜2 期間)

- **整合済**: レベル 2 以上 (見た目 80% 一致)
- **部分整合**: レベル 0 or 1 (構造 / 動線のみ)
- **判定方法**: PR レビュアー (Claude Code / 開発者) が mockup スクショと実装スクショを並べて目視比較

### 将来 (Tier 3 完了後)

- **整合済**: レベル 3 (95% 一致、自動 visual regression で diff 0)
- **判定方法**: Chromatic / Percy 等の visual regression テスト

## PR テンプレートでの記入要件

各 PR で以下を明記する (R-29 写経駆動開発):

```markdown
## 整合性レベル

本 PR で達成した整合性レベル: **[レベル N]**

### 視覚比較 (R-29 5 段階)

- [ ] mockup の該当 jsx component を完全 Read 済 (file: `<path>`、行範囲: `L<start>-L<end>`)
- [ ] mockup の該当画面スクショを Read 済 (`docs/mockups/v1.0/screenshots/<id>.png`)
- [ ] 既存 RN 実装ファイルを Read 済 (R-18)
- [ ] 実装後、RN スクショを撮影 (実機 / Web / アタッチメント)
- [ ] mockup スクショと RN スクショを並べて Read で目視比較

### 一致項目チェック (レベル 2 達成のため)

- [ ] 写真/画像のサイズ
- [ ] フォント (NotoSerif / NotoSans / IBMPlexMono)
- [ ] 色トークン (BRAND*GREEN / BG*\_ / TEXT\_\_)
- [ ] 主要レイアウト (3 階層 / 2 階層 / Hero + Tabs 等)
- [ ] 主要ボタン/CTA の配置
```

## ADR-0020 §画面マップ との関係

ADR-0020 §Notes §画面マップ の各 row の「種別」列で本レベルを参照:

| 種別                           | 意味                                  |
| ------------------------------ | ------------------------------------- |
| 「整合済 (PR #N)」             | レベル 2 以上、PR #N で達成           |
| 「部分整合 (PR #N、レベル 0)」 | レベル 0 のみ達成、本格整合は別 Issue |
| 「部分整合 (PR #N、レベル 1)」 | レベル 1 達成、視覚整合は別 Issue     |
| 「大改修」                     | 未着手                                |
| 「不採用」                     | 採用しない (例: WateringGraphScreen)  |
