# ja 文言スタイルガイド

> 対象: `src/core/i18n/locales/ja.ts` の全キー (ユーザー可視の日本語文言)。
> 採択経緯: `docs/adr/ADR-0060-ja-copy-style-standard.md`。
> 機械検査: `pnpm i18n:style` (`scripts/i18n-style-ja.mjs`)。禁止語は `pnpm i18n:forbidden` が正 (R-3 / constraints 第5章)。

## 基準 (10 項目)

1. **説明文 (`*Desc` / `*Body` / `*Detail`) = です・ます調の文なら文末に句点「。」**
   例: `入力された内容は保存されません。`。短いキャプション (カード副題等) は体言止め + 句点なしでも可。同じ UI 役割内ではどちらかに揃える。
2. **タイトル・ラベル・ボタン (`*Title` / `*Label` / `*Cta` / `*Badge` / `*Action`) = 体言止め or 短い動詞句、句点なし**
   例: `バックアップを作成` / `変更を破棄しますか?` (確認タイトルの疑問形は可)
3. **英数字と和文の間に半角スペース。和文同士の間にはスペースを入れない**
   例: `Pro にアップグレード` / `盆栽 1 つにつき {count} 枚まで`。プレースホルダ `{...}` も英数字扱い (`{count} 件`)。`登録すると、 ここに` のような句読点後の半角スペース (折返しヒント残骸) は不可。
4. **記号は半角**: `?` `!` `(` `)`。全角の `？！（）`・全角英数字・全角スペースは使わない。
5. **表記揺れ辞書で統一**: 買い切り (買切 ✕) / ください (下さい ✕) / できます (出来ます ✕) / いただき (頂き ✕)。辞書は `scripts/i18n-style-ja.mjs` の `NOTATION_DICT` が正、監査で随時追加。
6. **簡潔に**: 1 文 1 情報。「〜することができます」→「〜できます」。冗長な前置きを書かない。ただし破壊的操作・バックアップ系の説明は安心材料なので削らない。
7. **専門用語 (剪定・施肥 等の盆栽ドメイン標準語) は維持**。初心者向け補足が必要な場合のみ 1 行併記 (R-5)。
8. **命令調・押し付けを避ける** (ADR-0011「診断しない、記録する」)。「〜すべき」「〜しましょう」(強い勧誘) は不可。ガイド文脈の「〜してみましょう」「〜できます」(提案形) は可。
9. **訳語の一貫性**: 同じ概念には同じ語を使う (例: 通知 = notification、「リマインダー」禁止は R-3)。固有名詞 (BonsaiLog / Pro / Free) は翻訳禁止リスト (ADR-0033 D3) が正。
10. **数値・単位の書式**: `{count} 件` `{count} 枚` のように数値と助数詞の間に半角スペース。金額は `¥9,800` (¥ と数字は密着)。

## 機械検査の範囲

`pnpm i18n:style` が検査するのは 3 / 4 / 5 と、suffix で判定可能なキーの 1 / 2 (句点ルール) のみ。6〜9 は人間 (Claude) レビューで担保し、PR テンプレ + `/i18n-add` Skill のチェックで運用する。
