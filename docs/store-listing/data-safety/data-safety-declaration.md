# Google Play Data Safety 宣言 — BonsaiLog

> **対象**: Google Play Console アプリ → ポリシーとプログラム → アプリのコンテンツ → Data Safety  
> **位置付け**: 配信前に必ず登録する宣言内容のソース (この docs から Play Console に転記)。  
> **根拠**: ADR-0017 §⑥ Data Safety + `docs/reference/constraints.md` §1-2 (ローカル保存) / §1-3 (PII 取得しない) / §2-3 (AdMob 運用ルール)

---

## 0. アプリ概要 (Data Safety コンテキスト)

BonsaiLog は **Local-first** の盆栽記録アプリ。ユーザー登録なし、クラウドバックエンドなし、AI 機能なし、推奨機能なし、位置情報非取得 (USDA Hardiness Zone レベルすら取得しない)。すべてのユーザーデータは端末内 (SQLite + AsyncStorage + SecureStore + 写真ファイル) に保存。

外部送信は以下 2 系統のみ:

- **RevenueCat** (課金管理、F-13): 匿名 UUID v4 (`$RCAnonymousID:` プレフィックス) のみ送信、PII なし
- **Google AdMob** (広告配信、F-14、Free のみ): 広告 ID、UMP 同意取得後のみ

---

## 1. Data Safety Questionnaire 回答

### 1.1 アプリは必須のユーザーデータ収集 / 共有を行いますか?

**Yes** — 課金履歴を RevenueCat (決済プロセッサ) と共有、デバイス ID を Google AdMob (広告ネットワーク、Free プランのみ + UMP 同意取得時) と共有。詳細は §2 参照。

### 1.2 アプリで収集するすべてのユーザーデータは転送中に暗号化されていますか?

**Yes** — すべての通信が HTTPS / TLS (RevenueCat / AdMob 公式 SDK が強制)。

### 1.3 ユーザーがデータの削除をリクエストする方法を提供していますか?

**Not applicable** — サーバー側にユーザーデータは保存されない。すべてのデータは端末内に保存され、以下で削除可能:

1. アプリ内で個別の盆栽 / 写真 / 作業ログを削除 (30 日ゴミ箱経由)
2. アプリをアンインストール (端末上の全データを削除)
3. サーバー側に削除すべきデータは存在しない (= 削除リクエストフォーム不要)

### 1.4 ユーザー登録は必要ですか?

**No** — メール / パスワード / 氏名 / 電話番号 / 個人情報を一切取得しない (constraints §1-3)。

## 2. データ種別の内訳

### 2.1 位置情報 (Location)

- **収集**: ❌ なし (USDA Hardiness Zone を含めて一切取得しない、ADR-0011 「記録のみ」哲学)
- **共有**: なし
- **目的**: なし
- **転送中暗号化**: N/A (取得していない)
- **必須 / 任意**: N/A

> Repolog との差分: Repolog は「Optional (ユーザー有効化時のみ位置情報をレポートに付与)」を宣言。BonsaiLog は ADR-0011 で位置情報非取得を確定。

### 2.2 写真と動画 (Photos and videos)

- **収集**: ✅ あり (カメラ / ライブラリから盆栽の写真をユーザーが追加)
- **共有**: なし — 端末内のみ保存
- **目的**: アプリ機能 (盆栽の写真記録、F-08)
- **転送中暗号化**: N/A (端末外へ送信しない)
- **必須 / 任意**: 必須 (コア機能、Free プランは 1 樹あたり 3 枚制限、Pro は無制限)

### 2.3 個人 / プロフィール情報 (Personal info)

- **収集**: ❌ なし
- **共有**: なし
- 補足: ユーザー登録なし、氏名 / メール / 電話番号 / 住所 / 生年月日のいずれも取得しない

### 2.4 メッセージ / コンタクト (Messages, Contacts, Calendar)

- **収集**: ❌ なし

### 2.5 課金履歴 (Purchase history)

- **収集**: ✅ あり (RevenueCat がサブスクリプション / 買い切りの状態を管理、F-13)
- **共有**: ✅ あり — RevenueCat (決済プロセッサ)
- **目的**: サブスクリプション管理 (Pro エンタイトルメント判定、Restore Purchases)
- **転送中暗号化**: ✅ Yes (HTTPS、RevenueCat SDK が強制)
- **必須 / 任意**: 任意 (Pro 機能加入時のみ)
- 補足: ユーザー紐付けは匿名 UUID v4 のみ (`$RCAnonymousID:` プレフィックス)、PII を一切送信しない (ADR-0009 §49)

### 2.6 デバイス ID または他の ID (Device or other IDs)

- **収集**: ✅ あり (AdMob、Free プランのみ + UMP 同意取得時)
- **共有**: ✅ あり — Google AdMob (広告ネットワーク)
- **目的**: 広告配信 (パーソナライズ広告 / non-personalized 広告)
- **転送中暗号化**: ✅ Yes (HTTPS)
- **必須 / 任意**: 任意 (Free プランのみ + UMP 同意で「許可」選択時)
- 補足: Pro プランは AdBanner 自体マウントしないため、デバイス ID 取得は発生しない (ADR-0010 / ADR-0017 §⑧)

### 2.7 アプリの操作 / アクティビティ (App activity, App info and performance)

- **収集**: ❌ なし (将来 Sentry 配線時に「クラッシュ診断」のみ収集、現状は未配線)
- **共有**: なし
- 補足: Sentry 配線時に本ファイル §2.7 を更新

### 2.8 健康 / フィットネス / 金融 / ファイル / 連絡先 / カレンダー

- **収集**: ❌ なし

## 4. 削除ポリシー (Data deletion)

ユーザーは以下の方法で全データを削除できる:

1. **アプリ内で個別削除**: 盆栽 / 写真 / 作業ログを 30 日ゴミ箱経由で削除 (F-02 / F-08)
2. **F-11 お引っ越し機能でエクスポート → アプリ再インストール**: 任意のタイミングで端末データをクリーン化
3. **アプリのアンインストール**: 端末上の全データ (SQLite / AsyncStorage / SecureStore / 写真ファイル) を完全削除
4. **サーバー側に削除すべきデータは存在しない**: BonsaiLog はクラウドバックエンドを持たないため、削除リクエストフォーム不要
